## 1. Summary

This PR delivers three personalization-focused additions on top of v2: (1) reverts the `sc-` slash-command prefix that was added in v2 — the Claude Code marketplace auto-namespaces commands as `/<plugin-name>:<command>` (e.g. `/bytheslice:prd-generator`), making the explicit prefix redundant; (2) adds a declarative `bytheslice.config.json` config layer so users can override per-agent model tiers, MCP availability, visual-review tooling, stage shape, and HITL categories without forking the plugin; (3) adds a new `bootstrap` skill (Stage 0) that scaffolds a fresh Next.js single-app or Turborepo monorepo via the official scaffolders, drops in a `bytheslice.config.json`, and creates a gitignored `ROADMAP.local.md` for personal future-version notes.

## 3. Breaking changes from v2.0

- **`/sc-X` slash commands removed.** Users who hand-typed `/sc-prd-generator` need to switch to `/bytheslice:prd-generator` (published form) or `/prd-generator` (bare form). Triggers in SKILL.md frontmatter include both forms so most autoinvocation continues to work.

That's the only breaking change. Everything else is additive.

---

## 4. New required project setup steps

None for existing projects. Two optional new entry points for new projects:

- `/bytheslice:bootstrap` — scaffold a brand-new project (skip if you already have one on disk)
- `bytheslice.config.json` at the project root — personalize defaults (entirely optional; the plugin works fine without it)
